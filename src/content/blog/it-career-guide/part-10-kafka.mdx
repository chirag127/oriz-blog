## 1. Why Kafka in 2026?

Apache Kafka powers the data infrastructure of Netflix, LinkedIn, Uber, and every major tech company. Understanding Kafka is essential for backend and distributed systems roles, with engineers commanding **20-30% higher salaries**.

## 2. Kafka Fundamentals

### Core Concepts

```
Producer → Topic → Partition → Consumer Group → Consumer
```

- **Producer**: Publishes messages to topics
- **Topic**: Categorizes messages into streams
- **Partition**: Enables parallel processing and scalability
- **Consumer Group**: Group of consumers sharing workload
- **Consumer**: Subscribes to partitions and processes messages

### Architecture

```java
// Java producer example
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);

ProducerRecord<String, String> record = new ProducerRecord<>("users", "user1", "{\"name\": \"John\"}");
producer.send(record);
producer.close();
```

## 3. Python Kafka Implementation

### Producer

```python
from kafka import KafkaProducer
import json

producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8')
)

def send_user_event(user_id, event_type, data):
    producer.send('user-events', {
        'user_id': user_id,
        'event_type': event_type,
        'data': data,
        'timestamp': time.time()
    })

# Async send
future = producer.send('orders', {'product': 'laptop', 'price': 999})
record_metadata = future.get(timeout=10)  # Block until sent
```

### Consumer

```python
from kafka import KafkaConsumer

consumer = KafkaConsumer(
    'user-events',
    bootstrap_servers=['localhost:9092'],
    value_deserializer=lambda m: json.loads(m.decode('utf-8')),
    group_id='user-service-group'
)

for message in consumer:
    print(f"Received: {message.value}")
    # Process message
    process_user_event(message.value)
```

### Consumer Configuration

```python
consumer = KafkaConsumer(
    'orders',
    bootstrap_servers=['localhost:9092'],
    group_id='inventory-service',
    auto_offset_reset='earliest',  # Read from beginning
    enable_auto_commit=True,
    auto_commit_interval_ms=1000,
    value_deserializer=lambda m: json.loads(m.decode('utf-8'))
)
```

## 4. Partitioning Strategies

### Partition Key Selection

```python
# Good partition keys - evenly distribute load
producer.send('orders', key=user_id.encode(), value=order_data)

# Bad partition keys - hotspots
producer.send('orders', key=b'constant', value=order_data)
```

### Custom Partitioner

```python
def custom_partitioner(key, partitions):
    # Custom logic for partition selection
    if key:
        return hash(key) % partitions
    return 0

producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    partitioner=custom_partitioner
)
```

### Partition Distribution

| Strategy | Use Case | Example |
|----------|----------|---------|
| **Hash-based** | Even distribution | `hash(key) % partitions` |
| **Round-robin** | No key required | Distribute evenly |
| **Range-based** | Sequential ordering | Time-series data |

## 5. Consumer Groups

### Group Management

```python
# Multiple consumers in same group share partitions
consumer1 = KafkaConsumer('orders', group_id='processing-group')
consumer2 = KafkaConsumer('orders', group_id='processing-group')

# Each partition assigned to one consumer in group
# Consumer1: partitions 0, 2
# Consumer2: partitions 1, 3
```

### Rebalancing

```python
from kafka import KafkaConsumer

consumer = KafkaConsumer(
    'orders',
    group_id='processing-group',
    on_commit_ready=lambda offsets: print(f"Committed: {offsets}"),
    on_join_group=lambda: print("Joined group")
)

# Handle rebalancing
for message in consumer:
    try:
        process_message(message)
        consumer.commit()  # Manual commit
    except Exception as e:
        consumer.commit_async()  # Async commit
```

## 6. Exactly-Once Semantics

### Idempotent Producer

```python
producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    enable_idempotence=True,  # Prevent duplicates
    acks='all',  # Wait for all replicas
    retries=5
)
```

### Transactional Producer

```python
from kafka import KafkaTransactionalProducer

producer = KafkaTransactionalProducer(
    bootstrap_servers=['localhost:9092'],
    transactional_id='inventory-tx-1'
)

try:
    producer.begin_transaction()
    producer.send('orders', order_data)
    producer.send('inventory', inventory_update)
    producer.commit_transaction()
except Exception as e:
    producer.abort_transaction()
```

### Exactly-Once Processing

```python
consumer = KafkaConsumer(
    'orders',
    group_id='exactly-once-group',
    enable_auto_commit=False
)

producer = KafkaTransactionalProducer(
    bootstrap_servers=['localhost:9092'],
    transactional_id='processing-tx-1'
)

for message in consumer:
    producer.begin_transaction()
    try:
        # Process message
        result = process_order(message.value)
        
        # Send to output topic
        producer.send('processed-orders', result)
        
        # Commit offset
        consumer.commit()
        producer.commit_transaction()
    except Exception as e:
        producer.abort_transaction()
```

## 7. Kafka Streams

### Stream Processing

```java
StreamsBuilder builder = new StreamsBuilder();

KStream<String, Order> orders = builder.stream("orders");

KStream<String, EnrichedOrder> enriched = orders
    .join(
        builder.stream("products"),
        (order, product) -> enrich(order, product),
        Joined.with(Serdes.String(), orderSerde, productSerde)
    )
    .filter((key, enriched) -> enriched.getAmount() > 100);

enriched.to("high-value-orders");

KafkaStreams streams = new KafkaStreams(builder.build(), props);
streams.start();
```

### Word Count Example

```java
KStream<String, String> textLines = builder.stream("text-lines");

KStream<String, Long> wordCounts = textLines
    .flatMapValues(text -> Arrays.asList(text.toLowerCase().split("\\W+")))
    .groupBy((key, word) -> word)
    .count()
    .toStream();

wordCounts.to("word-counts");
```

## 10. Common Kafka Interview Questions

| Question | Answer |
|----------|--------|
| **Difference between Kafka and RabbitMQ?** | Kafka is a distributed commit log, RabbitMQ is a message broker. Kafka persists messages, RabbitMQ is ephemeral. |
| **What is a consumer group?** | Group of consumers sharing partitions for fault tolerance and scalability. |
| **How to handle backpressure?** | Use consumer throttling, increase partitions, or implement batch processing. |
| **What are Kafka Streams advantages?** | Embedded processing, exactly-once semantics, no external processing framework needed. |
| **Difference between at-least-once and at-most-once?** | At-least-once may duplicate, at-most-once may lose messages. Exactly-once guarantees no duplicates. |
