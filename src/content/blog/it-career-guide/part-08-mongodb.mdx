## 1. Why MongoDB in 2026?

MongoDB remains the leading NoSQL database for modern applications, offering flexibility, scalability, and developer-friendly JSON-like documents.

### Key Advantages
- **Flexible schema**: Documents can have different structures
- **Horizontal scaling**: Sharding for massive scale
- **Rich query language**: Aggregation pipeline
- **Multi-cloud**: MongoDB Atlas managed service
- **Real-time**: Change streams for event-driven apps

### Salary Impact
Engineers with MongoDB expertise command **10-20% higher salaries** in backend and data engineering roles.

## 2. MongoDB Basics

### Document Structure

```javascript
// Sample document
{
  "_id": ObjectId("507f1f77bcf86cd799439011"),
  "name": "John Doe",
  "email": "john@example.com",
  "age": 30,
  "address": {
    "street": "123 Main St",
    "city": "New York",
    "state": "NY"
  },
  "orders": [
    {"id": 1, "total": 99.99, "date": ISODate("2024-01-15")},
    {"id": 2, "total": 149.99, "date": ISODate("2024-02-20")}
  ],
  "createdAt": ISODate("2024-01-01T00:00:00Z")
}
```

### Basic CRUD Operations

```javascript
// Create
db.users.insertOne({name: "Alice", email: "alice@example.com"})

// Read
db.users.find({age: {$gte: 18}})
db.users.findOne({email: "alice@example.com"})

// Update
db.users.updateOne({email: "alice@example.com"}, {$set: {age: 25}})

// Delete
db.users.deleteOne({email: "alice@example.com"})
```

## 3. Schema Design Patterns

### Embedding vs Referencing

#### Embedding (Denormalization)

```javascript
// When to embed:
// - One-to-few relationships
// - Data accessed together
// - Limited document growth

// User with embedded orders
{
  "_id": ObjectId("..."),
  "name": "John",
  "orders": [
    {"id": 1, "product": "Laptop", "price": 999},
    {"id": 2, "product": "Mouse", "price": 29}
  ]
}
```

#### Referencing (Normalization)

```javascript
// When to reference:
// - One-to-many relationships
// - Large documents
// - Independent lifecycle

// Users collection
{
  "_id": ObjectId("user1"),
  "name": "John"
}

// Orders collection
{
  "_id": ObjectId("order1"),
  "userId": ObjectId("user1"),
  "product": "Laptop",
  "price": 999
}

// Using $lookup for joins
db.orders.aggregate([
  {$lookup: {
    from: "users",
    localField: "userId",
    foreignField: "_id",
    as: "user"
  }}
])
```

### Schema Design Guidelines

| Pattern | Use Case | Example |
|---------|----------|---------|
| **Embedded Documents** | Hierarchical data, limited growth | Blog post with comments |
| **Document References** | Many-to-many, large datasets | E-commerce with products/categories |
| **Hybrid Approach** | Mix of both | User profile with recent orders |

## 4. Aggregation Pipeline

### Pipeline Stages

```javascript
// Complex aggregation example
db.orders.aggregate([
  // Match specific criteria
  {$match: {status: "completed", date: {$gte: ISODate("2024-01-01")}}},
  
  // Group by user
  {$group: {
    _id: "$userId",
    totalSpent: {$sum: "$amount"},
    orderCount: {$sum: 1},
    avgOrderValue: {$avg: "$amount"}
  }},
  
  // Filter groups
  {$match: {totalSpent: {$gte: 1000}}},
  
  // Sort results
  {$sort: {totalSpent: -1}},
  
  // Limit output
  {$limit: 10}
])
```

### Common Aggregation Operators

```javascript
// Accumulators
{$sum: "$field"}          // Sum values
{$avg: "$field"}          // Average
{$max: "$field"}          // Maximum
{$min: "$field"}          // Minimum
{$push: "$field"}         // Add to array
{$addToSet: "$field"}     // Add unique values

// Operators
{$cond: {if: "$price", then: 10, else: 5}}  // Conditional
{$mergeObjects: ["$fields", {"new": "value"}]}  // Merge objects
```

### Change Streams

```javascript
// Watch for changes in real-time
const changeStream = db.orders.watch([
  {$match: {operationType: "insert"}}
]);

changeStream.on("change", (change) => {
  console.log("New order:", change.fullDocument);
});
```

## 5. Indexing Strategies

### Single Field Index

```javascript
db.users.createIndex({"email": 1})
db.users.createIndex({"createdAt": -1})  // Descending
```

### Compound Index

```javascript
// Order matters in compound indexes!
db.users.createIndex({"status": 1, "createdAt": -1})

// Queries that use this index:
db.users.find({"status": "active"})
db.users.find({"status": "active"}).sort({"createdAt": -1})

// Won't efficiently use index:
db.users.find({"createdAt": -1})  // Missing status field
```

### Text Index

```javascript
db.articles.createIndex({"title": "text", "content": "text"})

// Text search
db.articles.find({$text: {$search: "mongodb database"}})
```

### Geospatial Index

```javascript
db.locations.createIndex({"location": "2dsphere"})

db.locations.find({
  location: {
    $near: {
      $geometry: {type: "Point", coordinates: [-73.9857, 40.7484]},
      $maxDistance: 1000
    }
  }
})
```

### Index Analysis

```javascript
// Explain query execution
db.users.find({"email": "test@example.com"}).explain("executionStats")

// Check index usage
db.users.find({"status": "active"}).explain("executionStats")
```

## 6. Performance and Scaling

### Replica Sets

```javascript
// 3-node replica set configuration
// Primary - accepts writes
// Secondary - replicates data
// Arbiter - breaks election ties

// Connection string
mongodb://primary:27017,secondary:27017/arbiter:27017/mydb?replicaSet=rs0
```

### Sharding

```javascript
// Enable sharding
sh.enableSharding("mydb")

// Shard collection
sh.shardCollection("mydb.users", {"email": "hashed"})

// Shard key considerations
// - High cardinality
// - Even distribution
// - Frequently queried field
```

### Atlas Managed Service

```javascript
// Connection string format
mongodb+srv://:<password>@cluster0.abc123.mongodb.net/mydb?retryWrites=true&w=majority

// Security features
// - Network isolation
// - Encryption at rest/in transit
// - Auditing
// - Backup and restore
```

## 8. Common MongoDB Interview Questions

| Question | Answer |
|----------|--------|
| **Difference between MongoDB and MySQL?** | MongoDB is document-based NoSQL, MySQL is relational SQL. MongoDB scales horizontally, MySQL vertically. |
| **What is a capped collection?** | Fixed-size collection that overwrites oldest documents when full. |
| **How to improve query performance?** | Create appropriate indexes, use explain(), optimize queries. |
| **What is replication?** | Multiple copies of data across servers for high availability. |
| **Difference between count() and countDocuments()?** | count() is deprecated, countDocuments() counts documents matching query. |
